# TCP Connection Issues

**Persona**: System Admin / IT Support

Resolve network connectivity failures between the Hub and paired mobile devices.

### Prerequisites
- Paired devices attempting to connect via the local network.

### Visual Workflow
> [DIAGRAM]: Check Hub IP -> Ping from Mobile -> Verify Port 8080 -> Establish NSP Mesh

### Step-by-Step Instructions
1. Open the **Command Prompt** on the Hub workstation.
2. Type `ipconfig` to verify the Hub's local IP address.
3. Ensure the Mobile device is on the same subnet (e.g., both on `192.168.1.x`).
4. Check if the Noxis Hub is listening on **Port 8080**.
5. Temporarily disable the Windows Firewall to test if it is blocking the connection.

### Example Solution
> [EXAMPLE]:
> **Issue**: Mobile says "Connection Refused"
> **Fix**: Disable Windows Firewall for Private Networks or add an Exception Rule for `NoxisHub.exe` on Port 8080.

### Troubleshooting
- **Network Isolation**: Many modern Wi-Fi routers have "AP Isolation" enabled; disable this to allow devices to talk to each other.
- **Port Conflict**: If another application is using port 8080, change the **NSP Port** in `settings.json`.
