# Managing the Product Registry (SKUs)

The SKU (Stock Keeping Unit) is the fundamental data unit in the Noxis Hub. Accurate SKU management is critical for financial integrity and inventory accuracy.

## Data Structure
Every SKU in the Noxis ecosystem requires:
- **Unique Code**: A non-repeating identifier (e.g., FAB-WH-001).
- **Tax Category**: Determines the GST/VAT rate applied during sales.
- **Base Unit**: The unit of measurement (Kg, Meters, Pieces).
- **Industrial Attributes**: Weight, material composition, and batch sensitivity.

## Adding a New SKU

1. **Initiate Entry**
   Navigate to **Inventory > SKU Registry** and click **"Add New SKU"**.

2. **Define Metadata**
   Enter the product name and category. 
   > [TIP]: Use consistent naming conventions (e.g., [Material]-[Color]-[Size]) for better searchability.

3. **Configure Pricing**
   Set the **Cost Price** (weighted average) and **Retail Price**.
   [EXAMPLE]: Cost: 500, Retail: 750, Margin: 33.3%

4. **Barcode Generation**
   If the product has an existing barcode, scan it now. Otherwise, click **"Generate"** to create a Noxis internal code.

5. **Set Thresholds**
   Define the **Low Stock Alert** level. The Hub will trigger a Sentinel Alert when stock falls below this value.

## Bulk Import
For large registries, use the **CSV/Excel Import** tool. Ensure your headers match the Noxis Schema exactly to prevent alignment errors in the financial ledger.
